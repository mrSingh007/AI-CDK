# Card Component

## Purpose
Provide a reusable card container for AI chatbot interfaces and related UI surfaces.

## What It Should Do
- Provide a flexible structure for header, body, and footer content.
- Support rendering dynamic application data in any card section.
- Work as a general surface for grouped information and actions.

## Customization Expectations
- Allow visual customization through style tokens and CSS custom properties.
- Support theme-level changes such as colors, borders, radius, and spacing.
- Support interaction variants such as non-clickable and clickable card behavior.

## AI Chatbot Usage
- Display message context, summaries, references, or actions in chatbot flows.
- Present dynamic content blocks returned from AI responses.
- Serve as a composable container in chat layouts where content shape may vary.

## Non-Goals
- Not a full data table or layout framework.
- Not responsible for fetching or transforming business data.
