# Introduction

### Welcome to the AI CDK documentation!

The `@ai-cdk/ui` packages provide AI building block components. Components are fully customizable to support your app design system and AI workflows.

AI CDK helps you build production-ready AI interfaces faster by combining reusable UI components with protocol-friendly rendering patterns.

## What you can build

- Chat interfaces with structured tool outputs.
- Human-in-the-loop approval and feedback flows.
- Dynamic payload-driven UI rendering from agent/tool responses.
- Composable AI features that still match your product design system.

## Cdk Packages

- `@ai-cdk/ui`: UI components for AI-first interfaces.
- `@ai-cdk/chat`: WIP, upcoming fully featured chat interfaces and protocol services.

## Design principles

- Component-first: compose small focused building blocks.
- Protocol-ready: map tool/payload data into UI with minimal glue code.
- Theme-friendly: customize through CSS variables and component APIs.
- Accessible by default: keyboard and screen reader support are first-class.

## Documentation roadmap

- `Getting Started/Installation`: environment setup and first install.
- `Getting Started/AI-Protocol Examples`: AG-UI and Vercel-AI protocol integration examples.
- `UI/*` and `Chat/*`: component-level API docs and usage stories.
